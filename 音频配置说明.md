## ⚙️ 修改配置

### 配置文件位置

**主要配置文件：** `src/config/audioConfig.ts`

### 配置选项

#### 选项1：使用本地文件（当前配置）

```typescript
{
  id: '1',
  audioUrl: '/audio/meditation-1.mp3',  // 本地文件路径
}
```

#### 选项2：使用在线URL

```typescript
{
  id: '1',
  audioUrl: 'https://your-server.com/audio/meditation.mp3',  // 在线URL
}
```

#### 选项3：修改文件名

如果您想使用不同的文件名：

```typescript
{
  id: '1',
  audioUrl: '/audio/我的冥想音频.mp3',  // 改为您的文件名
}
```

## 📝 文件要求

- **格式**：MP3（推荐）或 WAV
- **采样率**：44.1kHz 或更高
- **比特率**：128kbps 或更高
- **文件大小**：建议不超过 10MB

## ⚠️ 常见问题

### Q: 音频无法播放？
A: 
1. 检查浏览器控制台的错误信息
2. 确认音频文件是否存在
3. 检查文件路径是否正确
4. 系统会自动尝试使用备用URL

### Q: 如何知道使用的是本地文件还是在线音频？
A: 打开浏览器控制台（F12），查看日志信息

### Q: 可以添加更多音频吗？
A: 可以！在 `src/config/audioConfig.ts` 的 `audioConfigs` 数组中添加新配置

### Q: 如何修改备用音频URL？
A: 编辑 `src/config/audioConfig.ts` 中的 `fallbackAudioUrls` 对象
