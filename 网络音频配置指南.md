## ⚠️ 重要注意事项

### 1. CORS跨域问题

网络音频URL必须支持跨域访问，服务器需要设置CORS头：

```
Access-Control-Allow-Origin: *
或
Access-Control-Allow-Origin: https://your-domain.com
```

### 2. HTTPS协议

- 推荐使用 HTTPS 协议（更安全）
- 如果网站使用 HTTPS，音频URL也应该是 HTTPS
- 否则浏览器可能会阻止混合内容

### 3. 音频格式

- 推荐使用 MP3 格式（兼容性最好）
- 也可以使用 WAV、OGG 等格式
- 确保浏览器支持该格式

### 4. URL稳定性

- 确保音频URL稳定可用
- 避免使用临时链接
- 建议使用CDN或云存储服务

## 🐛 故障排除

### 问题1：音频无法加载

**可能原因：**
- URL不正确或文件不存在
- CORS跨域问题
- 网络连接问题

**解决方案：**
1. 在浏览器中直接访问URL，确认可以访问
2. 检查控制台的错误信息
3. 确认服务器支持CORS

### 问题2：CORS错误

**错误信息：**
```
Access to audio at 'https://...' from origin '...' has been blocked by CORS policy
```

**解决方案：**
- 联系服务器管理员配置CORS头
- 或使用支持CORS的CDN服务
- 或将音频文件放到自己的服务器

### 问题3：音频格式不支持

**解决方案：**
- 使用MP3格式（兼容性最好）
- 或提供多种格式的备用URL
